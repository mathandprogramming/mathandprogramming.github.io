Common Encoding Methods

Posted on October 13, 2019

Before going over the encoding methods, why do we need encoding? Common reasons are:

  • Most machine leaning algortihm only takes numerical inputs but categorical inputs can be some type of string such as {male, female}.

  • Even if categorical feature have numeric data, they do not have numerical property like numerical data. For example, the zip code of Chelsea, NYC is 10001 and the zip code of Lower East Side, NYC is 10002. However, the sum of two zip codes, 10001 + 10002 = 20003, doesn’t mean anything.

There are many different encoding methods when it comes to categorical data for a supervised machine learning algorithm. Here, I organize some of most common methods. Most common encoding methods:

  • Label encoding: this method changes every categorical value to a number.
  • One-hot encoding: this method takes each category value and turns it into a binary vetor whose length is the number of distinct values in category and all elements in the binary vector are zero except one element that specifies the category.
  • Feature hashing: this method also vectorizes each category value (like one hot encoding). It works by applying a hash function to the features and using their hash values as indices directly.
  • Mean target encoding: This method encodes categorical value with the mean of the target variable.

When to use each encoding method:

  • Label encoding: this simple encoding can be used when the purpose of encoding is only to change non-numeric categorical data to numerical data.
  • One-hot encoding: this can be used if the feature is nominal data which means there are no order among them.
  • Feature hashing: this can be used in the same case as in One-hot encoding and the number of distinct categories in the feature is large.
  • Mean target encoding: this cen be used if we want to bring out the relation between similar categories and the target itself.
